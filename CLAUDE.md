# Career Intelligence Assistant

App that analyzes a CV against N job postings. FastAPI + SQLite backend in `backend/`, React SPA in `frontend/`. See `plan.md` for the full design; the short version:

- A **Run** is the unit of history: one CV + N jobs at a point in time.
- Deterministic work (skill overlap, scoring) is computed in plain Python — never generated by an LLM. The LLM is used only for extraction and narrative judgment.
- The original upload is kept on disk (`data_dir`); only `parsed_json` is stored in the DB. PDFs go to the LLM as raw files (vision handles scans/layout); DOCX text is extracted locally (`python-docx`, capped by `max_docx_chars`) and never persisted — so re-running extraction just re-reads the file.

## Backend (`backend/`)

### Stack

- Python 3.13, FastAPI, async SQLAlchemy 2.0 + `aiosqlite`
- `pydantic-settings` for config (`configs/settings.py`, loaded from `.env`)
- OpenAI (`openai` SDK, `client.responses.parse` structured outputs): cheap model for extraction/adjudication, stronger model for narrative
- `bge-small-en-v1.5` embeddings in-memory (numpy) — no vector DB
- `uv` for dependency management (`uv add`, `uv run`)
- Run: `cd backend && uv run main.py` (port 8000)

### Data model (`backend/models/`)

`Resume` → `Run` → (`Job`, `JobResult`). All tables carry `id` + `created_at` (via `TimestampMixin`). JSON columns hold parsed extraction and computed diff/narrative output.

## Frontend (`frontend/`)

### Stack

- Vite SPA: React 19 + TypeScript, Tailwind CSS v4 (`@tailwindcss/vite`, CSS-first — no `tailwind.config.js`/`postcss.config.js`, only `@import "tailwindcss"` + `@theme` in `src/index.css`)
- react-router (unified package — do **not** install `react-router-dom`), TanStack Query v5
- `@` alias → `src/`. Run: `cd frontend && npm run dev` (port 5173)
- `src/api/` is the only place that talks to the backend: `client.ts` (fetch wrapper + `ApiError`), `types.ts` (mirrors `backend/schemas/` 1:1, snake_case, ISO datetime strings), `keys.ts` (query-key factory)
- UI philosophy: modern, minimalistic, simple UX, as readable as possible

### API access / CORS

The backend has **no CORS middleware** — keep it that way. Dev uses the Vite proxy (`/api` → `http://localhost:8000` in `vite.config.ts`); prod should stay same-origin (reverse proxy in compose). Don't add `CORSMiddleware` reflexively.

## Code style

- **Minimal comments.** Let type hints and clear names carry the meaning. Add a comment only where logic is genuinely non-obvious (a formula, a workaround, a subtle invariant).
- **Docstrings only when the class/method is complex** enough to need one. Skip them on obvious CRUD, simple helpers, and self-explanatory models.
- **Type hints everywhere.** Python: `list[X]`, `dict[K, V]`, `X | None`. TypeScript: strict mode, no `any`.
- Prefer pure functions for deterministic logic (scoring, skill matching) — testable without an API key.
